# ADR-0001: Use Supabase as the Global Knowledge DB

> **Status:** Proposed
> **Date:** 2025-08-27
> **Related Issue:**  (Phase 1 setup)
> **Design Doc:** None

## Context
We need a relational + vector/graph-capable backbone for knowledge assets and lineage.

## Decision
Use Supabase (Postgres) for relational schema, pair with pgvector, and maintain pointers
to external vector/graph services if needed.

## Consequences
- Centralized schema and lineage
- Low friction for SQL + functions
- Clear upgrade path to managed Postgres
