# ADR-0002: Simplified Content Data Entry - Entity or Note Only

> **Status:** In Progress
> **Date:** 2025-08-30
> **Design Doc:** None — small mixed ADR

---

## Executive Summary

Collapses the content data entry model from a three-way routing system (Entity/Concept/Note) to a binary choice (Entity or Note), eliminating ambiguity between "1P concepts" and "entities."

---

## Context

Initially designed a content data entry system with complex routing logic:
- Capture table routed to either Entities table OR Concepts table
- Separate handling for "entity" vs "concept" types
- Multi-select field conversion to JSON
- Complex field mappings for different target tables

During implementation, we identified a fundamental issue: the distinction between "1P concepts" and "entities" created unnecessary complexity and ambiguity.

---

## Decision

1. Collapse the data model to two categories: **Entity** (all concrete deliverables, content, and 1P concept definitions) and **Note** (quick captures, thoughts, references, preliminary ideas).
2. 1P concepts that warrant promotion become entities with `content_kind='concept'`.
3. Aligns with the single aggregate root pattern (Entity as the primary business entity).

---

## Consequences

**Positive:**
- Clearer mental model for content creators
- Simplified automation logic
- Reduced data duplication and sync issues
- Better alignment with domain-driven design principles

**Negative:**
- Requires migration of existing concept records
- Some loss of metadata specificity for pure reference concepts
